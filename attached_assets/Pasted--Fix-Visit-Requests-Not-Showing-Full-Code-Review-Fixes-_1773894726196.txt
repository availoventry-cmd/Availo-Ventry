## BUG 1 (CRITICAL): Visit requests sorted wrong — newest at bottom, so new requests are invisible

In `artifacts/api-server/src/routes/visitRequests.ts` line 39, the query uses `.orderBy(visitRequestsTable.createdAt)` which defaults to ASC (oldest first). New visit requests go to the bottom and may not appear in the first page of results.

### Fix
Change line 39 from:
```typescript
.orderBy(visitRequestsTable.createdAt);
```
to:
```typescript
.orderBy(sql`${visitRequestsTable.createdAt} DESC`);
```

Add `sql` to the drizzle-orm import at the top of the file:
```typescript
import { eq, and, sql } from "drizzle-orm";
```

## BUG 2: Search box on Visit Requests page does nothing

The backend GET handler destructures `search` from query params (line 34) but never uses it for filtering. The frontend sends `search` but nothing gets filtered.

### Fix
Replace the entire GET `/` handler in `artifacts/api-server/src/routes/visitRequests.ts` with:

```typescript
// GET /api/organizations/:orgId/visit-requests
router.get("/", requireAuth, requireOrgAccess, async (req, res) => {
  try {
    const { orgId } = req.params;
    const { status, type, branchId, dateFrom, dateTo, search, page = "1", limit = "20" } = req.query;
    const user = req.user!;

    let requests = await db.select().from(visitRequestsTable)
      .where(eq(visitRequestsTable.orgId, orgId))
      .orderBy(sql`${visitRequestsTable.createdAt} DESC`);

    // Role-based filtering
    if (user.role === "host_employee") {
      requests = requests.filter(r => r.hostUserId === user.id);
    }
    if (user.role === "receptionist") {
      const today = new Date().toISOString().split("T")[0];
      requests = requests.filter(r => r.scheduledDate === today && (user.branchId ? r.branchId === user.branchId : true));
    }
    if (user.role === "visitor_manager" && user.branchId) {
      requests = requests.filter(r => r.branchId === user.branchId);
    }

    if (status) requests = requests.filter(r => r.status === status);
    if (type) requests = requests.filter(r => r.type === type);
    if (branchId) requests = requests.filter(r => r.branchId === (branchId as string));
    if (dateFrom) requests = requests.filter(r => r.scheduledDate >= (dateFrom as string));
    if (dateTo) requests = requests.filter(r => r.scheduledDate <= (dateTo as string));

    const p = parseInt(page as string);
    const l = parseInt(limit as string);

    // If search is provided, enrich ALL results first then filter by visitor name/phone/email/purpose
    if (search && (search as string).trim()) {
      const s = (search as string).toLowerCase().trim();
      const allEnriched = await Promise.all(requests.map(enrichRequest));
      const matched = allEnriched.filter(r =>
        r.visitor?.fullName?.toLowerCase().includes(s) ||
        r.visitor?.phone?.includes(s) ||
        r.visitor?.email?.toLowerCase().includes(s) ||
        r.purpose?.toLowerCase().includes(s)
      );
      const total = matched.length;
      const paginated = matched.slice((p - 1) * l, p * l);
      res.json({ data: paginated, meta: { total, page: p, limit: l, totalPages: Math.ceil(total / l) } });
      return;
    }

    const total = requests.length;
    const paginated = requests.slice((p - 1) * l, p * l);
    const enriched = await Promise.all(paginated.map(enrichRequest));
    res.json({ data: enriched, meta: { total, page: p, limit: l, totalPages: Math.ceil(total / l) } });
  } catch (err) {
    console.error("List visit requests error:", err);
    res.status(500).json({ error: "Internal Server Error" });
  }
});
```

Make sure `sql` is imported from `drizzle-orm` at the top of the file.

## BUG 6: Org admin role-based filtering may hide walk-in requests

In the visit requests GET handler, org_admin is NOT filtered — they see ALL org requests. But visitor_manager is filtered by `user.branchId`. If a visitor_manager has no branchId assigned, they see all requests (which is correct). However, if they DO have a branchId, walk-in requests submitted via the public booking page to a different branch won't show. This is working as intended, but document this behavior.

No code change needed — just be aware that org_admin sees everything, but branch-assigned roles only see their branch.

When the request is later approved via the approval endpoint in `visitRequests.ts`, the approval handler does NOT generate a QR code — it just changes status to "approved". So the visitor pass page shows no QR code even after approval.

Change the approve handler's update logic. Find the `.set({...})` call and update it to:

```typescript
// First get the current request to check if QR exists
const currentReq = await db.select().from(visitRequestsTable)
  .where(and(eq(visitRequestsTable.id, requestId), eq(visitRequestsTable.orgId, orgId)))
  .limit(1);

if (!currentReq[0]) {

const qrCode = currentReq[0].qrCode || generateQrCode();
const trackingToken = currentReq[0].trackingToken || generateToken(16);

await db.update(visitRequestsTable).set({
```

## BUG 8: Visitor data not updated on repeat visits

In `publicRoutes.ts` lines 61-80, when a visitor with the same phone already exists, the code reuses the existing visitor record but does NOT update their name, email, or company. So if a visitor registers again with an email (which they didn't provide the first time), the email is lost.

### Fix
After finding an existing visitor (line 62), update their info if new data is provided:

```typescript
if (!visitor) {
  // Create new visitor (existing code)
  ...
} else {
  // Update visitor with any new info provided
  const visitorUpdates: Record<string, unknown> = { updatedAt: new Date() };
  if (visitorName && visitorName !== visitor.fullName) visitorUpdates.fullName = visitorName;
  if (email && !visitor.email) visitorUpdates.email = email;
  if (nationalId && !visitor.nationalIdNumber) visitorUpdates.nationalIdNumber = nationalId;
  if (companyName && !visitor.companyName) visitorUpdates.companyName = companyName;
  
  if (Object.keys(visitorUpdates).length > 1) {
    await db.update(visitorsTable).set(visitorUpdates as Partial<typeof visitorsTable.$inferInsert>)
      .where(eq(visitorsTable.id, visitor.id));
  }
}
```
